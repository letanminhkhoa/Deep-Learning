1. Problem and dataset exploration (EDA)

Analyzing the multimodal dataset and the Weak Supervision labeling method.

Problem statement

Objective

Multimodal models require cross-modal alignment between Text and Image semantic spaces. The primary task is to classify images into 5 major semantic groups: animal, vehicle, person, food, indoor.

Dataset Highlights

Caption Processing & Labeling

  • Dataset source: yerevann/coco-karpathy via HuggingFace Datasets.
  • Labeling via Captions: We defined specific keyword mapping for the 5 classes. The get_label_from_captions function scans the original text captions to automatically assign class labels (Weak Supervision).
  • Images are downloaded directly from URLs, aggressively filtering out corrupted links to curate 800 clean samples.

3. Model building, evaluation, and comparison

Comparing the Zero-shot inference capabilities versus prototype-based Few-shot learning of CLIP.

Constructing the Reference Space

Zero-shot: Pass the prompt template "a photo of a {class}" through CLIP's Text Encoder to obtain reference semantic vectors.
Few-shot: Compute the mean of image feature vectors for each class in the Training set (Prototype Learning), establishing the class centroid reference vector.

Computing Similarity

Calculate the dot product between the Test Image Features and the Text/Prototype Features. Because both vectors are L2-normalized, the dot product mathematically equals Cosine Similarity.

TTA Voting Mechanism

During inference, each test image possesses 7 distinct feature vectors (from the 7 crops). We perform similarity matching for all 7 instances independently and select the most frequently predicted label (Majority Voting) as the final outcome.

Assign1_Multimodal.ipynb (Few-shot Prototypes) 
# Build prototypes for Few-shot Learning
prototypes = []
for c in range(num_classes):
    class_feats = X_train_fixed[y_train == c]
    proto = class_feats.mean(axis=0)
    proto = proto / (np.linalg.norm(proto) + 1e-8)
    prototypes.append(proto)
prototypes = np.array(prototypes)

# Predict with TTA Mean Aggregation
fs_predictions = []
for feats in X_test:
    feats = feats / np.linalg.norm(feats, axis=1, keepdims=True)
    logits = feats @ prototypes.T
    logits_mean = logits.mean(axis=0)
    pred = np.argmax(logits_mean)
    fs_predictions.append(pred)

Performance Summary

Zero-shot vs Few-shot

The Few-shot method outperforms the Zero-shot approach because it leverages actual labeled support samples from the dataset to adjust the feature space closer to the COCO distribution, rather than relying solely on generalized text alignment.

Error Analysis

Explaining the Discrepancies

  • Noisy Labels: Labels are auto-generated via Keyword Matching from captions. For example, an image displaying both a person and a dog might be captioned focusing on the "dog", thus mislabeling it as 'Animal' and confusing the model's metrics.
  • Class Confusion: Significant overlap and confusion consistently occur between the 'animal', 'vehicle', and 'person' classes due to complex visual compositions.

Test-Time Augmentation (TTA)

Enhancing Robustness

A crucial extension for extra credit: Instead of evaluating an image only once, TTA allows the model to synthesize information from multiple spatial perspectives (Crops) of the exact same image. Applying Majority Voting across these crops mitigates noise and yields far more accurate predictions.
